# AI早报数据源可信度评估

## 当前数据源

### Tier 1: 高可信度（优先保留）

| 源 | 可信度 | 特点 | 垃圾信息风险 |
|---|---|---|---|
| **Hacker News** | ⭐⭐⭐⭐⭐ | 技术社区讨论，有投票机制，评论区质量高 | 低。HN用户群体专业，炒作内容容易被downvote |
| **Solidot** | ⭐⭐⭐⭐ | 中文开源/科技资讯，报道偏硬核 | 低。偏向技术事实，较少营销内容 |
| **GitHub Trending** | ⭐⭐⭐⭐⭐ | 实际代码活跃度，无编辑主观性 | 极低。纯数据驱动，无法造假 |

### Tier 2: 中等可信度（谨慎使用）

| 源 | 可信度 | 特点 | 垃圾信息风险 |
|---|---|---|---|
| **少数派** | ⭐⭐⭐ | 中文科技媒体，质量尚可但有商业推广 | 中。部分内容是软文/推广，需要辨别 |
| **Product Hunt** | ⭐⭐⭐ | 产品发布平台，有Launch炒作文化 | 高。Launch当天有大量营销行为，真实用户反馈需等3-7天后 |
| **36氪** | ⭐⭐ | 商业科技媒体，融资新闻多 | 高。很多PR稿，创业者自我宣传 |
| **机器之心** | ⭐⭐ | AI垂直媒体，技术解读但有时浮夸 | 中。标题党多，内容深度参差不齐 |

### Tier 3: 低可信度（不推荐）

| 源 | 可信度 | 问题 |
|---|---|---|
| **量子位** | ⭐ | 标题党严重，夸大其词，经常炒作 |
| **微博热搜** | ⭐ | 商业化严重，热搜可买，情绪化内容多 |
| **今日头条AI频道** | ⭐ | 算法推荐导致信息茧房，低质量重复内容 |

## 可信度判断标准

### ✅ 高可信度信号
1. **可溯源**：链接到原始论文/公告/代码
2. **有时间戳**：明确标注发布时间
3. **有作者/机构署名**：非匿名来源
4. **技术细节具体**：提到具体版本号、参数、架构
5. **社区验证**：HN/Reddit上有技术讨论

### ❌ 垃圾信息信号
1. **标题含"震惊"/"重磅"/"颠覆"**：情绪煽动型
2. **无具体时间**：模糊表述如"近日"
3. **无限定词**："业界首个""全球领先"无具体比较对象
4. **只有融资新闻无产品**：PPT公司特征
5. **同一内容多平台重复**：PR稿铺量特征

## 建议调整

### 保留（Tier 1）
- Hacker News: 4-5条/天
- Solidot: 2-3条/天  
- GitHub Trending: 2-3条/天

### 可选（Tier 2，人工过滤）
- 少数派: 1-2条/天（只保留非推广内容）
- Product Hunt: 1-2条/天（只保留有真实用户评论的）

### 放弃（Tier 3）
- 36氪/机器之心/量子位：垃圾信息比例过高，人工过滤成本高
- 微博热搜：情绪化、可购买、无法验证

## 真实性验证流程

对于每条候选新闻：

1. **交叉验证**：同一事件是否在HN/Solidot都有讨论？
2. **原始来源追溯**：点击链接，看是否指向官方公告/论文/GitHub
3. **时间检查**：是否3天内的旧闻被重新包装？
4. **评论质量**：HN评论是否有人指出问题？
5. **实时搜索验证**：用web_search搜索该事件，看是否有其他媒体报道

## 观点获取策略

**大众观点**：
- 搜索 "{标题} Reddit/HN 讨论"
- 搜索 "{标题} 网友怎么看"
- 查看原帖评论区（HN/Solidot）

**反常识观点**：
- 搜索 "{标题} 深度分析"
- 搜索 "{标题} 争议/批评"
- 查看HN评论中最高赞的critical comment

## 结论

建议精简数据源到5个以内，宁可少而精，不要多而杂。

当前可靠的组合：
- Hacker News（海外技术视角）
- Solidot（中文硬核视角）
- GitHub Trending（代码/工程视角）

再增加2-3个高质量中文源即可达到15条/天目标。
