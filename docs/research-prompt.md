# AI早报数据源与内容质量优化 - 深度研究 Prompt

## 背景
我正在构建一个 AI 早报网站，目标是为抖音 AI 口播创作者提供高质量的每日 AI 行业热点。

**当前架构**：
- 数据源：RSS (Hacker News, Solidot, 少数派) + 尝试 Product Hunt/GitHub
- 内容字段：标题、发生了什么、为什么火、大众观点、反常识认知、相关热点
- 时间线：06:07 抓取 → 09:07 推送
- 目标：15-20 条高质量卡片/天

**已尝试但未满意的方案**：
1. ❌ 规则生成大众观点（基于关键词匹配）→ 太假，不够真实
2. ❌ kimi-search 插件实时搜索 → 插件调用不稳定
3. ❌ Product Hunt/GitHub → 反爬虫限制 (403)
4. ❌ 36氪/机器之心/量子位 → PR稿太多，可信度存疑

---

## 核心问题

### 问题 1: 如何高效获取真实的"大众观点"？

**需求**：对每条 AI 新闻，获取真实的大众讨论/观点

**约束**：
- 不能依赖单一 API（可能被封或不稳定）
- 需要可验证的观点（最好有来源链接）
- 每条新闻处理时间 < 30 秒（要在 3 小时内处理完 20 条）

**已排除方案**：
- Kimi Search API → 不稳定
- 规则生成 → 不真实
- Reddit API → 需要代理，风控严格

---

### 问题 2: 如何扩展到 15-20 条/天的可信数据源？

**当前来源**（7条左右）：
- Hacker News（海外技术）
- Solidot（中文硬核）
- 少数派（中文科技）

**需要的补充**：
- 高质量中文源（PR稿少，事实性强）
- 海外源（Twitter/X 上 AI 圈讨论、ArXiv 每日更新、TechCrunch/TheVerge AI板块）
- 不要：纯营销号、PR稿工厂、标题党

---

### 问题 3: 如何自动化获取"反常识认知"？

**定义**：正确但反直觉的观点，需要有专业深度

**示例**：
- 大众：OpenAI 技术领先不可超越
- 反常识：OpenAI 的壁垒是数据飞轮而非技术，用户习惯迁移成本低

**难点**：
- 需要专业判断，不能自动生成
- 可能需要多源对比（HN 评论区的 critical comment、专家分析）

---
